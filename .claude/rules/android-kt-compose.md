# Android / Kotlin / Compose Rule

Apply this rule when creating or modifying Android source code.

- Use Kotlin and Jetpack Compose only.
- Generate code that can be opened and validated in Android Studio.
- Prefer readability and explicitness over cleverness.
- Use stable package naming and consistent suffixes for ViewModel, UiState, UiAction, and Route.
- Keep composables focused and avoid hidden mutable behavior.
- Avoid global mutable state.
